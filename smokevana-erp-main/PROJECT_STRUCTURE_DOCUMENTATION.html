1. System Overview

What is SmokeVana ERP?

SmokeVana ERP is your complete business management solution that helps you run every aspect of your business from one centralized system. Whether you're managing products, tracking inventory, processing sales, handling purchases, or running online stores, everything is integrated and automated for maximum efficiency.

What Your System Can Do:

  • ✅ Manage Multiple Businesses & Locations - Run multiple stores or warehouses from one system
  • ✅ Complete Product Management - Organize products with categories, brands, variations, and flexible pricing
  • ✅ Point of Sale (POS) - Process in-store sales quickly and efficiently
  • ✅ Purchase Management - Handle supplier orders, receipts, and inventory updates automatically
  • ✅ Real-Time Inventory - Track stock levels across all locations in real-time
  • ✅ Customer & Supplier Management - Complete CRM to manage all your business relationships
  • ✅ Online Stores - Separate B2B (business customers) and B2C (end consumers) e-commerce platforms
  • ✅ Financial Management - Complete accounting with accounts, transactions, and financial reports
  • ✅ Restaurant Operations - Special features for restaurant and food service businesses
  • ✅ WooCommerce Integration - Connect with your existing WooCommerce store
  • ✅ Multiple Payment Options - Accept payments via Stripe, PayPal, Razorpay, and more

11. B2B Online Store Features

Overview

Your B2B (Business-to-Business) online store allows your business customers to shop online, place orders, track shipments, and manage their accounts. All features are accessible through secure API endpoints that can be integrated into websites or mobile apps.

Customer Account Management

Available Features:

  • ✅ Customer registration and login
  • ✅ Password reset functionality
  • ✅ Secure session management
  • ✅ Profile management

API Endpoints:

POST /api/customer/login
Allows business customers to log into their account using email and password. Returns authentication token for accessing protected features.
POST /api/customer/register
Creates a new business customer account. Requires business information, contact details, and password. Returns account confirmation.
POST /api/customer/forgot-password
Sends password reset link to customer's email address when they forget their password.
POST /api/customer/set-password
Allows customer to set a new password using the reset token received via email.
POST /api/customer/refresh-token
Refreshes the authentication token to extend the customer's session without re-login.
GET /api/customer/logout
Logs out the current customer and invalidates their authentication token.

Profile & Account Features

  • ✅ View and update customer profile
  • ✅ Receive notifications about orders and updates
  • ✅ Contact support team
  • ✅ Subscribe/unsubscribe to newsletters

API Endpoints:

GET /api/customer/my-account
Retrieves complete customer profile information including name, email, contact details, and account settings.
PUT /api/customer/update
Updates customer profile information such as name, email, phone number, and other personal details.
GET /api/customer/notification
Fetches all notifications for the customer including order updates, promotions, and system messages.
POST /api/customer/contact-us
Submits a contact form or support request to the business. Customer can ask questions or report issues.
POST /api/customer/subscribe
Subscribes the customer to receive newsletters, promotional emails, and marketing communications.
POST /api/customer/unsubscribe
Unsubscribes the customer from all marketing emails and newsletters.

Address Management

  • ✅ Save multiple shipping addresses
  • ✅ Set default shipping address
  • ✅ Add, edit, or delete addresses
  • ✅ Quick address selection during checkout

API Endpoints:

GET /api/customer/saved-addresses
Retrieves all saved shipping addresses for the customer. Useful for quick address selection during checkout.
POST /api/customer/create-saved-address
Creates and saves a new shipping address. Customer can add multiple addresses for different delivery locations.
GET /api/customer/get-saved-address/{id}
Gets details of a specific saved address by its ID. Used to view or edit a particular address.
PUT /api/customer/update-saved-address/{id}
Updates an existing saved address. Customer can modify address details like street, city, zip code, etc.
DELETE /api/customer/delete-saved-address/{id}
Permanently deletes a saved address from the customer's address book.
POST /api/customer/set-default-address/{id}
Sets a specific address as the default shipping address. This address will be pre-selected during checkout.
GET /api/checkout-address
Gets the address that will be used for the current checkout process. Returns default address if set.
POST /api/checkout-address
Sets or updates the shipping address for the current checkout session. Can be a saved address or new address.

Product Catalog & Browsing

  • ✅ Browse all products with filters (price, category, brand)
  • ✅ Search products by name or description
  • ✅ View detailed product information
  • ✅ Filter by categories and brands
  • ✅ View product reviews and ratings
  • ✅ Navigate through category and brand menus

API Endpoints:

GET /api/shop
Main shop page endpoint. Returns products with advanced filtering options (price range, category, brand, availability). Supports pagination.
GET /api/all-product
Retrieves all available products in the catalog. Useful for product listings or inventory displays.
GET /api/search
Searches products by name, description, or SKU. Returns matching products with relevance ranking.
GET /api/product/{slug}
Gets detailed information about a single product including images, variations, pricing, stock status, and specifications.
GET /api/product-list/{slug}
Returns a list of products based on a specific identifier or filter. Used for category or collection pages.
GET /api/category-product/{slugs}
Fetches all products belonging to a specific category. Supports nested categories using slug paths.
GET /api/multi-category/{slugs}
Retrieves products from multiple categories at once. Useful for showing products across related categories.
GET /api/brand-product/{slug}
Gets all products from a specific brand. Customers can browse products by their favorite brands.
GET /api/brand-category-menu
Returns navigation menu structure showing categories and brands. Used for building site navigation.
GET /api/brand-category-menu-cat2brand
Returns menu organized by categories first, then brands within each category. Alternative navigation structure.
GET /api/brand-list
Retrieves list of all available brands in the system. Used for brand filtering or brand showcase pages.
GET /api/product/{productId}/reviews
Gets all customer reviews and ratings for a specific product. Includes review text, ratings, and reviewer information.

Shopping Cart

  • ✅ Add multiple products to cart
  • ✅ Update quantities easily
  • ✅ Remove items from cart
  • ✅ Apply discount codes and promotions
  • ✅ Cart saved automatically
  • ✅ State-based product restrictions

API Endpoints:

GET /api/cart
Retrieves all items currently in the customer's shopping cart. Includes product details, quantities, prices, and totals.
POST /api/cart
Adds multiple products to cart or updates existing cart items in bulk. Efficient for adding many items at once.
POST /api/cart/reduce
Reduces the quantity of a specific item in the cart. If quantity reaches zero, item is removed automatically.
DELETE /api/cart
Removes a specific item from the shopping cart. Customer can delete unwanted products before checkout.
DELETE /api/cart/empty
Clears the entire shopping cart, removing all items. Useful for starting fresh or canceling a shopping session.
DELETE /api/cart/restricted/{itemId?}
Removes products that are restricted in the customer's state/location. Ensures compliance with geographic restrictions.
POST /api/cart/apply-discount
Applies a discount code or promotion to the cart. Validates the code and calculates discounted totals.
POST /api/cart/remove-discount
Removes an applied discount from the cart. Cart totals are recalculated without the discount.

Order Management

  • ✅ View all past orders and invoices
  • ✅ Track order status and delivery
  • ✅ Download and print invoices
  • ✅ Request returns for orders
  • ✅ Cancel pending orders

GET /api/customer/my-invoices
Gets all invoices issued to the customer. Includes paid and unpaid invoices with payment status.
GET /api/customer/my-order/{orderId}
Fetches detailed information about a specific order including all items, quantities, prices, shipping details, and status.
GET /api/customer/my-order-print/{orderId}
Generates and returns a printable PDF invoice for the order. Customer can download or print for records.
POST /api/customer/return-invoice
Submits a return request for an order. Customer can specify items to return and reason for return.
POST /api/sells/cancel/{id}
Cancels a pending sales order. Only works for orders that haven't been processed or shipped yet.
DELETE /api/sells/voidSell/{id}
Voids a completed sales transaction. Used for correcting errors or handling special cases. Requires proper permissions.

Payment Options

13. Unified B2C Store (Multi-Brand)

Overview

The Unified B2C store allows customers to browse all brands at a specific location in one unified catalog. This is perfect for multi-brand stores or marketplaces where you want customers to see everything available at a location.

How It Works

Location-Based Only: Customers browse by store location, seeing all brands together
Example: "Downtown Store" shows all products from all brands at that location
Unified Experience: Single catalog view for easier product discovery

Available Features

  • ✅ Browse all products at a location (all brands combined)
  • ✅ Search across all brands
  • ✅ Filter by categories
  • ✅ View products by specific brand (optional)
  • ✅ Unified shopping experience
