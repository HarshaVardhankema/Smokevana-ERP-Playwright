📋 Project Introduction

About SmokeVana ERP

SmokeVana ERP is a comprehensive Enterprise Resource Planning system built on Laravel 9.x framework. It's designed to manage all aspects of a business including product catalog, inventory, sales, purchases, customer relationships, e-commerce, accounting, and multi-location operations.

System Overview

The system is a multi-tenant, multi-location ERP solution that supports:

  • ✅ Multiple businesses and locationsSupports managing multiple business entities and their physical locations within a single system instance.
  • ✅ Product catalog with variations and pricingComprehensive product management system supporting product variations, SKUs, and flexible pricing strategies.
  • ✅ Point of Sale (POS) and sales managementComplete POS system for in-store transactions and comprehensive sales order management.
  • ✅ Purchase and procurement operationsFull purchase order workflow from requisition to receipt, including supplier management.
  • ✅ Inventory tracking across locationsReal-time inventory tracking with multi-location stock management and transfer capabilities.
  • ✅ Customer and supplier management (CRM)Unified contact management system for customers and suppliers with relationship tracking.
  • ✅ B2B and B2C e-commerce platformsSeparate e-commerce solutions for business-to-business and business-to-consumer transactions.
  • ✅ Accounting and financial managementComplete accounting module with chart of accounts, transactions, and financial reporting.
  • ✅ Multiple payment gateway integrationsSupport for various payment processors including Stripe, PayPal, Razorpay, and more.
  • ✅ WooCommerce synchronizationBidirectional sync with WooCommerce for products, orders, and customer data.
  • ✅ Restaurant management featuresSpecialized features for restaurant operations including table management and kitchen display systems.
  • ✅ Comprehensive reporting and analyticsExtensive reporting system covering sales, purchases, inventory, financials, and custom reports.

Technology Stack

Laravel 9.x PHP 8.0+ MySQL/MariaDB Blade Templates jQuery Vue.js Bootstrap Laravel Passport JWT Auth

Project Structure

The project follows Laravel MVC architecture with modular design:

  • app/Http/Controllers/ - All application controllersContains all HTTP controllers handling web and API requests, organized by feature modules.
  • app/ - Core models and business logicHouses Eloquent models, business logic classes, events, jobs, and core application components.
  • app/Utils/ - Utility classes (TransactionUtil, ProductUtil, etc.)Reusable utility classes providing common business operations like transaction creation and product management.
  • Modules/ - Modular extensions (WooCommerce, etc.)Separate modules for extended functionality like WooCommerce integration, following modular architecture.
  • database/migrations/ - Database schemaAll database migration files defining table structures, relationships, and schema changes over time.
  • resources/views/ - Blade templatesFrontend view templates using Laravel Blade templating engine for rendering HTML pages.
  • routes/ - Application routesRoute definition files for web routes, API routes, and specialized route groups like B2C unified routes.
⚠️ Important Notes for Developers:
  • All database changes must be done through migrationsNever modify database schema directly; use Laravel migrations to ensure version control and consistency across environments.
  • Always filter queries by business_id for multi-tenant supportCritical for data isolation; every query must include business_id to prevent cross-tenant data access.
  • Stock is tracked at variation level, not product levelInventory quantities are maintained per product variation (size, color, etc.), not at the parent product level.
  • Use TransactionUtil for all transaction operationsCentralized utility class ensures consistent transaction creation, stock updates, and accounting entries across the system.
  • Follow PSR-12 coding standardsAdhere to PHP-FIG PSR-12 coding standard for code consistency, readability, and maintainability.
  • Test on development environment before production deploymentAlways test changes thoroughly in development/staging environment to prevent production issues.
